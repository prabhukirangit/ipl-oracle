# IPL Oracle

A MiroFish-style swarm intelligence engine for IPL cricket match prediction — spawns ~60 autonomous agents per match and runs 100 parallel simulations to surface an emergent win probability.

## What It Does

IPL Oracle builds a digital cricket world that plays itself out via Monte Carlo simulation — each match request runs N independent ball-by-ball simulations (default 10–500 depending on mode, up to 50,000) with unique random seeds, then aggregates the results into win probabilities with confidence intervals:

- **MiroFish swarm engine** — one autonomous agent per match entity (every player, coach, stadium, pitch, weather system, crowd, umpire)
- **Monte Carlo simulations** — ball-by-ball T20 match loop run N times (up to 50,000) with unique random seeds, sampling outcomes from weighted probability distributions shaped by 50+ hidden institutional factors. Aggregate win probability + 95% CI from the full distribution. Persona/Hybrid: default 10, max 100 sims. Probabilistic: default 500, max 50,000 sims.
- **Hybrid LLM mode** — LLM reasoning fires at high-leverage moments via 5 explicit triggers: death overs (18+), wicket clusters (3+ down), close chases (<=40 in last 5 overs), post-wicket pressure balls, and pressure index >= 0.65. Routine balls use fast probabilistic sampling (~15-20% of balls use LLM). Built-in rate limiting: max 5 concurrent LLM calls with 200ms minimum spacing, plus retry with exponential backoff (2s/4s/8s) on 429 errors.
- **60+ agents per match** — PlayerAgent x 22+, CoachAgent x 2, CaptainAgent x 2, StadiumAgent, PitchAgent, WeatherAgent, CrowdAgent, UmpireAgent x 2, ReportAgent
- **Information-isolated agent decisions** — bowler and batsman decide concurrently via `asyncio.gather()`; neither sees the other's plan. Only the OutcomeResolver combines both decisions through a matchup matrix (slog vs yorker = wicket, pull vs bouncer = six, etc.)
- **Dynamic Field Interception** — CaptainAgent sets fielder positions per ball; boundaries must survive both pre-sample weight suppression (4+ deep fielders reduce boundary probability) and post-sample shot-to-fielder geometric interception (drive→long_off, pull→deep_square_leg, cut→deep_point). Applies in all three modes.
- **Live XI Cascade** — 4-level playing XI fetcher: IPLT20 (Playwright) -> Cricbuzz (stealth) -> Google News RSS -> local SQUAD_SEED. Supports partial results and fuzzy name matching. Web sources tried for IMMINENT/LIVE matches; FUTURE uses SQUAD_SEED.
- **Full Impact Player rule enforcement** — foreign cap hard law, 5-name substitute pool, optimal phase windows
- **Ball replacement rule 2026** — captain can request new ball after 10th over in night matches (once per innings)
- **Dew model** — wrist spinner RPM decay curves, finger spin degradation, over-by-over condition updates
- **18+ BallContext factors** — over rate penalties, intent shifts, spinner entry timing, drop catch variance, collapse contagion, anchor penalty, franchise clutch factor, post-timeout shifts, captain defensive tendency, lower-order independence, and more
- **CatBoost ML ensemble** — CatBoost win-probability model trained on 6 seasons of ball-by-ball IPL data (2020-2025). 16 numeric + 8 categorical features. Blended at 30% weight with simulation results (70%) across all 3 modes. Season-based train/val/test split (no data leakage). Graceful fallback if model unavailable or below AUC threshold.
- **Report Agent** — ReACT loop with tool-calling, win probability with 95% CI, score distribution histograms, aggregate winning factors (no player names — systemic factors only), bold scoreline prediction

## Architecture

```
Layer 1 — Data Scrapers + XI Cascade
    XI Cascade: IPLT20 -> Cricbuzz stealth -> News RSS -> SQUAD_SEED (4-level fallback)
    Playwright -> IPLT20 (schedule, confirmed XI, Impact Player pool)
    Playwright + playwright-stealth -> Cricbuzz (live scorecard, XI fallback)
    Google News RSS -> team/player news, XI text-mining
    httpx -> OpenWeatherMap (weather), Howstat (career records)
    Fuzzy name matching: scraped names -> SQUAD_SEED for profile building

Layer 2 — Agent System
    BaseAgent (memory via Mem0, LLM via Anthropic claude-sonnet / OpenAI / Gemini / local)
    ~60 agents spawned per match by AgentFactory
    Profiles built once (read-only); mutable state deepcopied per sim
    18+ BallContext factors injected mathematically per ball
    Bowler + batsman decide concurrently (information-isolated)
    OutcomeResolver: matchup matrix combines both decisions + field interception

Layer 3 — Monte Carlo Simulation Engine
    MatchEngine: ball-by-ball T20 loop, enhanced pressure index (7 factors)
    Monte Carlo: N independent sims with unique seeds → aggregate win probability + CI
    Hybrid LLM: 5 explicit high-leverage triggers (death overs, wicket clusters,
        close chases, post-wicket balls, pressure >= 0.65)
    ParallelRunner: batches of 25 via asyncio.gather(), 0.3s pause between
    Rate limiting: semaphore (max 5 concurrent) + 200ms min delay + exponential backoff (2s/4s/8s) on 429
    KuzuDB knowledge graph: head-to-head matchups, batter-bowler pairs

Layer 3.5 — CatBoost ML Ensemble
    CatBoost win-probability model: 16 numeric + 8 categorical features
    Trained on IPL 2020-2025 ball-by-ball data (season-based split)
    70/30 blend: simulation drives 70%, CatBoost historical prior 30%
    Graceful fallback: disabled if model missing or AUC < 0.70

Layer 4 — Report + API
    ReportAgent: aggregate synthesis, no player names, systemic factors only
    Win probability + 95% CI, score distribution, 3-5 concise bullet points
    Key Winning Factors: aggregated across all sims
    FastAPI: REST + WebSocket streaming of live sim events
    Vue 3 frontend: real-time progress, D3 charts, agent interview UI
```

## Quick Start

**Backend:**

```bash
cd ipl-oracle/backend
uv sync
uv run python run.py
```

> Chromium is auto-installed on first use — no manual `playwright install` needed.

**Frontend:**

```bash
cd ipl-oracle/frontend
npm install
npm run dev
```

The frontend dev server starts at `http://localhost:5173` and proxies API calls to `http://localhost:8000`.

## Configuration

Copy `.env.example` to `.env` in the project root and fill in:

| Key | Description |
|-----|-------------|
| `LLM_PROVIDER` | `anthropic` (default), `openai`, `gemini`, or `local` |
| `ANTHROPIC_API_KEY` | Required if `LLM_PROVIDER=anthropic` |
| `OPENAI_API_KEY` | Required if `LLM_PROVIDER=openai` |
| `GEMINI_API_KEY` | Required if `LLM_PROVIDER=gemini` |
| `LOCAL_LLM_BASE_URL` | For `local` provider — e.g. `http://localhost:11434/v1` (Ollama default) |
| `LOCAL_LLM_MODEL` | Model name for local provider — e.g. `qwen2.5:14b` |
| `LLM_MAX_CONCURRENT` | Max concurrent LLM calls (default: `5`) |
| `LLM_MIN_DELAY_MS` | Minimum delay between LLM calls in ms (default: `200`) |
| `LLM_PRESSURE_THRESHOLD` | Pressure index threshold for hybrid LLM trigger (default: `0.65`) |
| `OPENWEATHERMAP_API_KEY` | Weather conditions at venue (free tier works) |
| `MEM0_API_KEY` | Agent memory (optional — falls back to in-process store) |
| `FRONTEND_URL` | Frontend origin for CORS (default: `http://localhost:5173`) |
| `APP_HOST` | Backend bind host (default: `0.0.0.0`) |
| `APP_PORT` | Backend port (default: `8000`) |

### Fully Offline Mode (Ollama)

Run predictions with zero cloud API calls:

```bash
# 1. Install Ollama: https://ollama.com/download
# 2. Pull a model
ollama pull qwen2.5:14b

# 3. Set in .env
LLM_PROVIDER=local
LOCAL_LLM_BASE_URL=http://localhost:11434/v1
LOCAL_LLM_MODEL=qwen2.5:14b
```

Recommended models by VRAM: `qwen2.5:32b` (24GB+), `qwen2.5:14b` (10-16GB), `qwen2.5:7b` (8GB), `qwen2.5:3b` (CPU-only).

Any OpenAI-compatible server works (LM Studio, vLLM, etc.) — just point `LOCAL_LLM_BASE_URL` at it.

## CLI Prediction (No Frontend Required)

Run predictions directly from the terminal:

```bash
cd ipl-oracle

# Quick prediction (abbreviations work)
uv run python scripts/predict.py --team1 MI --team2 CSK

# Full team names, custom venue + pitch
uv run python scripts/predict.py --team1 "Mumbai Indians" --team2 "Chennai Super Kings" \
    --venue "Wankhede Stadium, Mumbai" --pitch batting --sims 100

# With LLM narrative (requires valid API key with credits)
uv run python scripts/predict.py --team1 SRH --team2 RR --llm

# JSON output (for piping to other tools)
uv run python scripts/predict.py --team1 KKR --team2 DC --json
```

**Options:**

| Flag | Default | Description |
|------|---------|-------------|
| `--team1` | (required) | Team 1 name or abbreviation (MI, CSK, RCB, KKR, SRH, RR, DC, GT, PBKS, LSG) |
| `--team2` | (required) | Team 2 name or abbreviation |
| `--venue` | team1 home | Venue name |
| `--date` | today 19:30 | Match date-time (ISO 8601) |
| `--pitch` | balanced | `batting`, `bowling`, `spin`, or `balanced` |
| `--sims` | 10 | Simulations to run (persona/hybrid: max 100, probabilistic: max 50000) |
| `--llm` | off | Enable LLM for narrative generation |
| `--json` | off | Raw JSON output |

## Training the CatBoost ML Model

The CatBoost win-probability model provides a 30% weighted historical prior that grounds simulation predictions in 6 seasons of real IPL data.

```bash
cd ipl-oracle

# Train on the included dataset (2020-2025 seasons)
uv run python scripts/train_catboost.py --data Dataset/IPL.csv

# Custom options
uv run python scripts/train_catboost.py --data Dataset/IPL.csv --iterations 2000 --verbose 50
```

**How it works:**
- Trains on ball-by-ball IPL data: 16 numeric features (run rate, pressure, phase, etc.) + 8 categorical features (teams, venue, players)
- Season-based split: train on 2020-2023, validate on 2024, test on 2025 (no temporal data leakage)
- If test AUC < 0.70, the model is disabled and simulation runs at 100% weight
- Model artifact saved to `backend/app/ml/models/win_prob.cbm` (git-ignored, ~50-200MB)
- Blending: `final_win_pct = 0.70 × simulation + 0.30 × CatBoost`

The model is automatically loaded at simulation startup. If the `.cbm` file is not present, the system falls back to simulation-only mode with no errors.

## Player Persona System

Each PlayerAgent is injected with a unique persona that shapes LLM decision-making in **Persona** and **Hybrid** modes. The persona is built from the following factors:

### Profile Stats (from SQUAD_SEED + career data)
- **Role** — batsman, bowler, allrounder, wicketkeeper
- **Batting style** — right-hand / left-hand bat
- **Bowling style** — pace, medium, off-spin, leg-spin, left-arm spin, etc.
- **Team** — current IPL franchise
- **Career stats** — strike rate, batting average, bowling economy, bowling average

### Personality Adjectives (per-player character description)
Every player in SQUAD_SEED carries a `personality` field — a comma-separated set of adjectives that describe their cricketing character. These are injected directly into the LLM system prompt as `"Your playing character: {personality}"`. Examples:

| Player | Personality |
|--------|-------------|
| Virat Kohli | intense, relentless, chase-master, feeds off pressure, never gives up |
| MS Dhoni | ice-cold finisher, legendary calm, helicopter shot, reads bowlers like books |
| Hardik Pandya | fearless, explosive, confident, swagger, big-match finisher |
| Jasprit Bumrah | yorker king, nerveless under pressure, unorthodox action, death-overs specialist |
| Rashid Khan | fearless leg-spinner, big hitter, Afghan warrior mentality, match-winner |

### Batting Personality (auto-generated from stats)
- **Approach** — `explosive_striker` (SR > 150, aggression > 0.7), `aggressive_accumulator` (SR > 140), `anchor` (avg > 35), or `situational`
- **Mental model** — narrative describing the player's batting mentality
- **Powerplay intent** — `aggressive` or `steady` based on aggression index
- **Death overs intent** — `calculated_aggression`
- **Chase mentality** — `calm_under_pressure` or `can_feel_pressure` based on pressure resilience trait

### Bowling Personality (for bowlers/allrounders)
- **Style** — pace type or spin variant
- **Approach** — economy-based classification (miserly < 7 eco, reliable < 8.5, attacking)
- **Death bowling specialist** — boolean from death overs specialization trait
- **Powerplay specialist** — boolean from powerplay specialization trait

### Communication Style
- **Tone** — `intense` / `focused` / `calm` based on aggression level
- **Encouragement pattern** — `aggressive_pump_up` or `quiet_support`
- **Under pressure talk** — `rally_teammates` or `focus_inward` based on big-match temperament

### How It Flows
```
SQUAD_SEED (personality adjectives)
    → AgentFactory.build_profile(personality=...)
        → PersonaLoader._generate_persona()
            → LLM system prompt: "You are {name}... Your playing character: {personality}. {mental_model}..."
                → PlayerAgent.bat_with_persona() / bowl_with_persona()
```
